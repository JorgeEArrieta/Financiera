Descripción

La aplicación financiera permite al usuario tener un registro de su economía diaria. Para eso, podrá crear registros de tipo ingreso, gastos, prestamos, ahorros. Asimismo, el usuario tendrá la opción de generar categorías diferentes para los ingresos y gastos que tenga. Lo que le permitirá tener una mayor visibilidad de su economía.

Menú - Ver

Ver

El menú "Ver" le permite al usuario acceder a la vista de los tipos de registro: Ahorro, Gasto tarjeta de crédito y préstamos.

Visualizar Ahorro

Dentro del formulario "visualizar ahorros" se podrá acceder al listado de registros de tipo ahorro que se encuentren dentro de la base de datos. También, dentro del mismo, el usuario podrá eliminar los registros que desee. Sin embargo, si el usuario creo el registro, con la opción que descuente del dinero disponible. El gasto generado deberá de ser eliminado de forma manual.

Visualizar Prestamo

El formulario "visualizar préstamo" le permite al usuario acceder a los diferentes prestamos creados. Para buscar los registros, se deberá de hacer click en el botón con una imagen de "lupa". Al hacerlo, se accederá a un submenú que trae un listado de los prestamos existentes.

Seleccionando el registro deseado, se podrá acceder a toda la información disponible sobre el mismo. Incluso, se podrá eliminar el registro con el botón con imagen de cesto de basura.

Gastos en tarjeta de crédito

Al acceder a este formulario, se tendrá acceso a los diferentes registros creados de este tipo. Con el botón con imagen de lupa, podremos seleccionar el gasto deseado.

Seleccionando el registro deseado, se podrá acceder a toda la información disponible sobre el mismo. Incluso, se podrá eliminar el registro con el botón con imagen de cesto de basura.

Ingresos

En el formulario "ingreso" le permitirá mantener un registro al usuario de sus haberes, pudiéndolos clasificar por tipo. El alta, requerirá de cuatro tipo de datos. El primero, el tipo de ingreso, permite catalogar el registro. Descripción, como su nombre lo indica, permite añadir una explicación al registro. Tercero, el ítem fecha, permite indicar la fecha en que se produjo el ingreso. El campo deberá tener el formato yyyy-mm-dd. Finalmente, el campo importe, como su nombre lo indica, permitirá ingresar el valor del ingreso. Es registro tomará como separador decimal el carácter ".".

Gastos

Dentro de este formulario, el usuario podrá ingresar los gastos. Para ello, deberá de completar cuatro registros. El primero, es el campo tipo de gasto. Donde el usuario podrá clasificar el mismo. El segundo, descripción, le permite agregarle un comentario u explicación al registro. Fecha, permite indicar cuando se realizó el gasto. Y, finalmente, el campo importe, nos permite registrar el valor del gasto.
Como en el formulario ingreso, el campo fecha, deberá de tener un formato yyyy-mm-dd. Para el campo importe, el separador decimal será ".".

Ahorro

Este formulario, nos permite llevar un registro sobre los ahorros, en diferentes tipos de divisas (pesos argentinos, euros, y dólares). Para realizar el proceso de alta, deberemos de ir completando un conjunto de campos. El primero, será el de moneda, en el cual, se tendrá que especificar el tipo de divisa del ahorro.
Segundo, el campo importe se ingresará el valor del ahorro. Acto seguido, indicaremos si al crear el registro, generamos automáticamente un registro de tipo gasto, por el monto del ahorro. Esto nos permite llevar un registro más preciso del dinero disponible. Vale aclarar que, si la divisa elegida para el proceso de alta, es el dólar o euro, el registro gasto generado será por el importe de la compra de la moneda. Para esto, antes, se debe de verificar la correcta parametrización de la moneda extranjera.
Luego, se deberá seleccionar si el ahorro genera tasa de interés, indicando, de ser verdadero, el porcentaje de la misma.
Finalmente, se podrá indicar si tiene una fecha de vencimiento. De ser así, se ingresará la fecha de la misma, con el formato yyyy-mm-dd. Si se especifica que el registro tiene una fecha de vencimiento, el programa verificara la fecha de vencimiento del registro cada vez que se ejecuta. De ser menor la fecha de caducidad a la fecha actual del sistema, se creará un registro de tipo "ingreso" por el valor total del registro. Eliminando, también, al ahorro del sistema.
Advertencia!Para poder utilizar el formulario de alta de ahorro, anteriormente, debe estar parametrizadas el valor de las divisas.

Gastos con tarjeta de crédito

El formulario "tarjeta de crédito", permite generar registros, de tipo gasto, pero con diferentes opciones. Para el proceso de alta, primero deberemos registrar la fecha en que se realizó el consumo. Al igual que el formulario "gasto", deberemos especificar el tipo y, luego, podremos ingresar una descripción del mismo. Luego, manteniendo la similaridad, el valor del consumo.
Las opciones que le siguen, son la cantidad de cuotas en que se va a realizar el pago del gasto. Manteniendo un máximo de 30 cuotas. Y si se aplica una tasa de interés, teniendo que especificar la misma. Cuando clickeamos en el botón guardar, el sistema generara un registro de tipo "gasto" por la cantidad de cuotas indicadas.

Prestamos / Créditos

Este formulario, permitirá realizar un alta de un préstamo. Para esto, se deberá especificar diferentes datos. Como la fecha, el importe del crédito y la cantidad de cuotas en que se deberá realizar el pago del mismo. Asimismo, se podrá especificar si el valor de las cuotas es un valor fijo o la tasa de interés del crédito.
Al generar el préstamo, se crearán dos tipos de registro; uno de tipo ingreso, y otro de gasto. El primero, contendrá el importe por el valor del crédito. En cambio, el de tipo ahorro, será uno por la cantidad de cuotas generadas. El cual contendrá el valor de la cuota y la fecha de pago.
